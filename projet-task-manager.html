Le Projet

Task Master Pro est une application web dynamique permettant de gérer ses tâches quotidiennes et ses projets complexes. L'objectif était de créer une solution plus robuste qu'une simple "To-Do List" en intégrant une base de données relationnelle, un système de tri avancé et une interface utilisateur moderne et persistante.

Objectif Pédagogique

Maîtriser le framework Flask, la gestion des requêtes HTTP (GET/POST), et l'interaction avec une base de données SQL brute.

Fonctionnalités Clés & Architecture

J'ai conçu l'application pour être à la fois performante côté serveur et réactive côté client.

  • Backend Python (Flask)
    Architecture MVC légère. Gestion des routes pour le CRUD (Créer, Lire, Mettre à jour, Supprimer) des tâches et des projets.
  • Base de Données (SQLite)
    Schéma relationnel liant les tables Tasks et Projects. Utilisation de requêtes SQL optimisées pour le filtrage.
  • Algorithme de Tri SQL
    Tri complexe combinant l'urgence (Priorité personnalisée via CASE WHEN) et la date d'échéance.
  • Interface Dynamique
    Utilisation d'accordéons pour les projets, de modales pour l'édition et du LocalStorage JS pour sauvegarder l'état des onglets et du Mode Sombre.

Focus Code : Le Moteur de Tri SQL

Une des parties les plus intéressantes a été de permettre le tri des tâches non seulement par date, mais aussi par "niveau d'urgence" (Urgent > Important > Normal), ce qui n'est pas un tri alphabétique standard. 

# Extrait de app.py - Logique de tri côté serveur
elif filtre == 'urgence':
    # On utilise CASE WHEN pour donner un "poids" à chaque niveau d'urgence
    sql_query += ''' ORDER BY CASE urgence 
                     WHEN 'danger' THEN 1 
                     WHEN 'warning' THEN 2 
                     WHEN 'primary' THEN 3 
                     ELSE 4 END ASC, date_echeance ASC'''

Cette approche permet de déléguer la logique de tri directement au moteur de base de données, rendant l'application plus rapide qu'un tri en Python.

Expérience Utilisateur (UX)

L'interface a été pensée pour réduire la friction utilisateur :

  • Mode Sombre/Clair
    Thème persistant grâce au stockage local du navigateur.
  • Mémoire d'État
    Si l'utilisateur rafraîchit la page, il reste sur le même onglet et garde ses dossiers projets ouverts.
  • Filtres Contextuels
    Possibilité de masquer les tâches terminées à l'intérieur de chaque projet.
