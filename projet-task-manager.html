Architecture & Backend

L'application repose sur le micro-framework Flask. Contrairement à une approche ORM classique (comme SQLAlchemy), j'ai fait le choix d'utiliser des requêtes SQL brutes via le module sqlite3 pour une maîtrise totale de la performance et de la structure relationnelle.

Modèle Relationnel

La base de données gère une relation One-to-Many entre les tables projects et tasks. Chaque tâche est liée à un projet via une clé étrangère (FOREIGN KEY), permettant une organisation structurée des données.

Tri SQL Avancé

Plutôt que de trier les données en Python (ce qui est lent sur de gros volumes), j'ai implémenté une logique de tri pondérée directement dans le SQL.

Extrait de app.py : Logique de pondération

Persistance UX

Utilisation intensive du localStorage en JavaScript. Si l'utilisateur ferme un onglet ou un dossier de projet, l'application se "souvient" de cet état au prochain chargement.

Data Visualization

Intégration de Chart.js pour générer des graphiques dynamiques (Doughnut & Bar) basés sur les données injectées par Jinja2.

Dark Mode

Implémentation d'un thème sombre complet via des variables CSS CSS variables (:root vs [data-theme="dark"]).

Focus Frontend : Gestion des onglets (script.js)

Ce script gère l'affichage dynamique sans rechargement de page et sauvegarde la préférence utilisateur. 

function showTab(event, tabId) {
    if(event) event.preventDefault();
    // Masquer toutes les sections
    document.querySelectorAll('.tab-section').forEach(el => el.classList.add('d-none'));
    
    // Afficher la cible avec animation
    const target = document.getElementById('tab-' + tabId);
    if(target) {
        target.classList.remove('d-none');
        target.classList.add('animate-stagger');
    }
    // Sauvegarde de l'état
    localStorage.setItem('activeTab', tabId);
}
